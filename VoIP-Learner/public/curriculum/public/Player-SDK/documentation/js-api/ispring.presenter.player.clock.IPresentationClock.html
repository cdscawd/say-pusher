interface
ispring.presenter.player.clock.IPresentationClock
The IPresentationClock interface provides information about the current playback position and the state of the presentation playback controller.

The IPresentationClock interface can be obtained using the clock() method on the IPresentationPlaybackController

interface.
See also
  • ispring.presenter.player.IPresentationPlaybackController#clock

Methods

buffering() → {boolean}

Indicates whether the player is buffering the data.
Returns:
Type: boolean

bufferStateChangeEvent() → (non-null) {ispring.events.IEventDispatcher}

Returns the IEventDispatcher interface dispatching the bufferStateChange event when the buffering state is changed.

The returned event dispatcher passes the presentation clock reference to the event handling function. See the following example:

    var clock; // IPresentationClock
    //...
    clock.bufferStateChangeEvent().addHandler(function(theClock){
        window.console.log("Clock buffering state has been changed to", theClock.buffering());
    });

Type: ispring.events.IEventDispatcher

running() → {boolean}

Indicates whether the clock is running.

The clock is running when it is started (see the started() method) AND the presentation is not being buffered or waiting for the user's action.

The suspended() and buffering() methods provide more precise information about the presentation clock state.

See also
  • ispring.presenter.player.clock.IPresentationClock#buffering
  • ispring.presenter.player.clock.IPresentationClock#started
  • ispring.presenter.player.clock.IPresentationClock#suspended
Returns:
Type: boolean

started() → {boolean}

Indicates whether the clock is started.

The started presentation clock does not run (see the running() method) if the presentation is being buffered or waiting for the user's action.
